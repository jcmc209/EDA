  • Objetivo

El objetivo principal de esta prctica es emplear tcnicas de Anlisis Exploratorio de Datos (EDA) para comprender a fondo un conjunto de datos relacionado con solicitudes de prstamos. Este anlisis busca identificar patrones y relaciones entre las variables que permitan predecir el comportamiento de los clientes respecto al cumplimiento o incumplimiento de sus pagos. En este sentido, se evaluarn tanto variables demogrficas como caractersticas financieras de los solicitantes, para determinar su influencia en la capacidad de devolver un prstamo. Este objetivo incluye el anlisis detallado de posibles correlaciones entre las variables, la imputacin de valores nulos, la transformacin de datos categricos y numricos, y el escalado de las caractersticas segn sea necesario.

En ltima instancia, el propsito es responder a la pregunta clave de negocio planteada: "Hay algn tipo de cliente ms propenso a no devolver un prstamo?". Esto servir como base para proponer estrategias que puedan mejorar la evaluacin de riesgo y reducir las tasas de incumplimiento en futuras decisiones de crdito.

  • Pasos

  1. Exploracin general de los datos: se analizan la distribucin de los datos y las relaciones entre las distintas variables en relacin a la variable objetivo.
  2. Correlaciones, valores nulos y outliers: se evaluan las correlaciones entre variables, se tratan los valores faltantes y los outliers. Se elaboran estrategias para su eliminacin o tratamiento.
  3. Codificacin y escalado: se codifican las variables categricas para poderlas emplearlas en el modelo. Se implementan las tcnicas de OneHotEncoding y TargetEncoding en funcin de los valores nicos de cada variable.

  • Caso de negocio

El caso de negocio gira en torno a un banco que se especializa en ofrecer distintos tipos de prstamos a sus clientes. Actualmente, la institucin enfrenta el desafo de identificar a los solicitantes que representan un mayor riesgo de incumplimiento. Para abordar este problema, se busca analizar el comportamiento histrico de los clientes que no han cumplido con sus pagos y establecer criterios claros para mitigar los riesgos en futuras operaciones de prstamo.

El anlisis exploratorio se convierte, en este contexto, en una herramienta clave para segmentar a los clientes segn sus caractersticas y establecer perfiles de riesgo ms precisos. Al responder preguntas como qu tipo de clientes o condiciones financieras estn ms relacionadas con el incumplimiento, el banco puede mejorar sus procesos de evaluacin crediticia.

def dame_variables_categoricas(dataset=None, max_unicos=100): 

    """ Se indica el valor de valores nicos mximo(max_unicos) como un parametro de manera que si lo escribimos a lo largo de la funcin
    ms de una vez solo se tenga que cambiar en una lnea.
    
    Inputs:
        -- dataset: pandas dataframe que contiene los datos
        -- max_unicos: mximo de unicos que se desean por columna(variable)
        """




    if dataset is None:
        print(u'\nFaltan argumentos por pasar a la funcin')
        return 1

    lista_variables_categoricas = []
    other = []

    for i in dataset.columns:
        unicos = int(len(np.unique(dataset[i].dropna(axis=0, how='all'))))
        if dataset[i].dtype in ['object', 'category'] or unicos < max_unicos: # A traves de la lista simplificamos el cdigo.
                lista_variables_categoricas.append(i)
        else:
                other.append(i)

    return lista_variables_categoricas, other
